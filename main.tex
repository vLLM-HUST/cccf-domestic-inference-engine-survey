\begin{cnabstract}
本文面向国产算力平台上的大模型推理系统，围绕执行架构、算子与图编译、KV Cache 管理、批处理调度、长上下文、多模态与结构化输出等关键问题，讨论国产芯片、编译栈、运行时和部署环境如何共同重塑推理引擎的设计边界。与通用 LLM serving 综述不同，本文不把相关工作铺展为功能清单，而是引入更贴近国内推理引擎建设任务的分析骨架：以统一抽象、统一指标和可观测底座为基础，以执行与通信、状态治理以及压缩与成本协同三条主路径为主线，分析平台异构、运行时状态复杂化与工程交付如何相互耦合，并据此将现有技术路线归纳为开源主干后端适配、平台一体化方案与混合式架构三类。进一步地，本文比较不同路线在生态兼容性、性能上限、复杂任务支持、运行时治理与长期维护成本上的差异，强调国产推理系统的关键不在局部算子是否足够快，而在平台契约、运行时组织和交付闭环能否同时成立。最后，本文结合 vLLM-HUST、vLLM-Ascend、MindIE、LMDeploy 等实践，讨论插件化平台能力、复杂请求状态管理、统一评测和交付控制面的后续方向，以期为国产推理系统的研发选型与架构演进提供更具可操作性的分析框架。
\end{cnabstract}

\noindent\textbf{关键词：}国产算力平台；大模型推理；推理引擎；运行时；软硬协同

\begin{enabstract}
This article surveys large-model inference systems on domestic computing platforms. Rather than offering another generic overview of LLM serving, it examines how hardware heterogeneity, runtime state complexity, and production-delivery constraints jointly reshape the design space of inference engines on domestic platforms. To better match the practical decomposition of domestic inference-engine construction, the paper uses an explicit analytical scaffold: a unified abstraction, metric, and observability base, together with three main paths for execution and communication, state governance, and compression-cost co-optimization. Under this scaffold, it reviews execution architecture, operators and graph compilation, KV cache management, batching and scheduling, long-context support, multimodal processing, and structured generation. It groups current technical routes into upstream-backend adaptation, vertically integrated stacks, and hybrid architectures, compares their tradeoffs in ecosystem compatibility, performance ceiling, complex-task support, runtime governance, and long-term maintainability, and argues that the key issue is not isolated kernel speed but whether platform contracts, runtime organization, and delivery workflows can evolve together. Based on systems such as vLLM-HUST, vLLM-Ascend, MindIE, and LMDeploy, it further outlines future directions including plugin-oriented platform contracts, complex-request runtime management, unified evaluation, and control-plane tooling for deployment and regression handling.
\end{enabstract}

\noindent\textbf{Keywords: }domestic computing platforms; large-model inference; inference engine; runtime systems; hardware-software co-design
