[[1]](#footnote-1)18世纪的哥尼斯堡有七座桥，把两岸和河中两块陆地连接在一起。人们好奇：能不能从某处出发，把所有桥各走一次，最后回到起点？欧拉给出的做法不是在街巷里兜转，而是把两岸与岛屿看成节点，把桥看成边，把现实地理抽象成一张只保留“谁与谁相连”的关系图[1]。问题随之变成：是否存在一条经过每条边且只经过一次的路径，如图1所示。答案是否定的，但更重要的是方法：先把关系抽出来，然后在“点—边”的世界里判断可能与不可能。这就是图论的出发点，也奠定了“用关系理解世界”的思维方式。

![图1](data:image/png;base64...)

图1 哥尼斯堡七桥问题

从七桥问题走到今天，图不仅是一种数学对象，更是一种贴近现实结构的表达。它强调连接而非形状，保留“谁影响谁、沿着什么路径影响、影响多强”这些本质关系。社交网络、交通与电网、分子成键与蛋白互作、传感器协同与时空事件，都呈现出节点多、强关系少且分布不整齐的特点。把这些对象图化，有两个直接好处：其一，表达贴近真实，不用把不规则的数据硬塞进规整网格；其二，焦点更聚焦，分析可以沿着关系本身展开，而不是在无关冗余上消耗算力。

这就引出了图计算。图计算不是单一算法，而是一整套“围绕关系来算”的流程：先做图建模，明确节点、边与权重的语义；再完成存取与划分，让图结构在内存或分布式系统中高效组织；随后选择计算模式（顶点/边/子图为中心的更新、匹配、检索、消息传递等）；最后落到并行执行，在多核、图形处理器（graphics processing unit, GPU）、集群上稳健扩展。它既包含最短路、连通分量、社区发现、子图匹配等经典方法，也包含能从结构中学习表征的图神经网络。

# 一、Graph for Science：图计算与科学计算

**图计算是处理稀疏关联关系的重要工具**[2][3] 现实数据如社交网络、知识图谱等，常常是“点多、边少且分布不整齐”，也就是稀疏而不规则的关联，如图2所示。

![图2](data:image/png;base64...)

图2 稀疏关系无处不在

图结构能够对此类关联关系进行清晰的表征与计算：其中，节点对应实体，边则刻画实体间的相互作用与依赖关系。该结构天然契合现实世界中的关系网络形态，尤其适用于表达非结构化、不规则的关联模式。相较于把数据硬排进规则矩阵，图上的并行与分布式执行更容易围绕顶点、边和子图来切分任务，既提升扩展性，也能跟随数据的动态变化而灵活伸缩。同时，图计算通过图神经网络等方法，能够学习和发现节点之间复杂的非线性关系，从而揭示潜在的模式和规律。这种能力对于从大规模数据中提取有意义的信息至关重要，尤其是在推荐系统、社交分析和生物信息学等领域。此外，图计算在处理稀疏关联关系时具有出色的扩展性，随着数据规模的增加，传统的计算方法往往难以满足需求，而图计算能够通过以图顶点/边/子图来进行任务划分，高效实现并行计算，能够更好地应对大规模稀疏数据的挑战[4]。这使得图计算在社交网络、推荐系统和生物信息学等领域的任务分析处理中表现出色。

**科学计算数据能利用图结构来表达**[5][6][7][9] 在众多科学场景中，许多数据天然适合用图结构来建模。如图3所示，在地球科学领域，通常采用“地-空-天”时空因果图、水域水系流动汇聚网、三角网数字高程模型等方式来表示气候气象、水文水利、地质地理等学科数据的潜在关联关系，构造复杂异构图数据；在高能物理、多体动力学等领域[5][6]，通常记录运动粒子的三维空间坐标[7]，通过计算空间距离来构造动态图数据；在天文观测[8]、生物演化等领域，科学数据通常是以序列的形态被采集，因此通过分析挖掘频繁子序列和相似子序列，构建其中隐藏的序列相似性，来构造图数据[9]。Graph for Science的主张是：先用图结构把科学对象之间的复杂关系明确表达出来，再用图计算去挖掘这些关系中的模式与规律，从而以高效方式处理海量稀疏数据并反哺科学计算与工程模拟。

# 二、图计算驱动的科学计算面临的挑战

在实践中，Graph for Science 的任务远不止把数据表示成点和线。要让图计算真正服务科学发现，通常要面临三大挑战。第一个挑战是数据：厘清哪些对象需要入图，什么才算有效的关系。第二个挑战是方法：让通用图算法与具体学科的机理和先验贴合。第三个挑战是计算：在海量、流式、异构的工程环境中把图上的工作负载跑得稳、跑得快。由此，挑战可归结为3个层面：数据形态、处理手段和计算模式。

**数据形态** 科学计算里的原始数据形态极为多样，很多并不是“天生就是图”。比如射电望远镜观测数据中，原始观测信号为“时间-频率”二维时序数据；高能物理粒子碰撞实验中，粒子与探测器发生作用的状态构成的事件树，单个事件包含空间位置、时间、角度、能量、动量、电荷量和粒子种类等；分子动力学模拟中，在一定反应条件下，分子的坐标、速度、力场等分子状态随时间变化达到稳态的过程。对于上述问题，如何定义谁是节点、什么是关系，从而把嘈杂的原始量测转换成有语义的图极具挑战性。这一步若处理不当，要么丢失关键信息，要么把噪声和偶然相关一并带入图里，后续分析就会走样。与此同时，这些数据又往往呈现“节点海量、连接稀疏”的结构特征[10][11]：无论是高能物理实验、银河恒星星图，还是蛋白质互作网络，平均度数远小于节点规模，说明用图表达契合其结构本质，但前提仍是把“数据如何图化”的语义说准、尺度拿稳。

**处理手段** 即使在分子、育种、蛋白这类“天然图”的场景里，通用的图算法与模型也未必直接可用，不同学科的约束与噪声模型差异很大，很难指望一套“放之四海皆准”的构图与处理流程。缺少贴合领域机理的算法与工具，常让分析效果打折。比较稳妥的做法是围绕具体科学任务抽象出“统一但领域化”的构图策略与计算模型，把几何、成键、能量守恒、时空因果等先验写进边的定义与权重设计，必要时引入多层、异构或动态的图表示；在算法上既利用传统的搜索、匹配、聚类与路径推理，也在合适问题中引入图神经网络等学习方法。这样，“数据—语义—算法”的链条才能彼此对齐，既保证可解释性，又把工程复杂度降低到可控范围。

**计算模式** 图一旦建好，主要挑战就转向了“如何把它算快、算稳”。首先，数据规模巨大且依赖复杂：科学场景中的图往往包含海量顶点和边，数据结构与依赖关系高度复杂，给实时处理带来巨大压力。其次，计算特征多样：科学工作流中并存着子图匹配、相似检索、路径搜索、图卷积等异质任务，单一计算类型的高效并行难以满足需求；不同计算类型在执行方式、通信模式、调度策略上差异显著，统一加速更具挑战[4][12][13][14]。最后，可靠性难以保障：大规模并行环境下，科学计算模式多样，故障与失效场景复杂，统一的容错机制设计困难，而低开销的容错又尤为关键。面对这些挑战，需要从“算法—系统—硬件”3个层面协同设计：在算法层通过图划分与任务编排消除不均衡，尽量将计算局部化；在系统层采用异步或流式执行减少全局等待，并把通信隐藏在计算背后；在硬件层则需要匹配合适的加速平台与数据布局，充分利用高带宽和并行度，同时兼顾能效与可扩展性。唯有将这三层环环相扣，图计算才能在实际科研生产线上跑得又稳又快。

## 3.1. 快速射电暴FRB搜寻

快速射电暴（fast radio burst, FRB）是一种宇宙中的高能天体物理现象，呈现瞬态电波脉冲，持续时间短、爆发能量大[15]。探测到更多FRB事件、扩充样本数量并丰富观测细节，将为研究其起源与性质提供极为重要的信息与线索。随着巡天数据量的不断增长，各大望远镜巡天项目都会面临海量数据处理的现实难题。传统的人工方法或者半自动化方法由于效率问题难以满足实际需要。基于海量天文观测数据开展FRB等自动化智能寻星任务已成为天文领域的热点。自动化搜寻多基于机器学习方法，例如多层感知机、决策树、深度神经网络等。这些方法属于有监督学习，需要依赖足量的标记样本；然而目前已知被探测到的FRB不足一千例，短时间内难以获得大量标注，且部分方法依赖特征工程，易受人为主观因素影响，从而对FRB搜寻造成偏差。

海量巡天数据本质上是FRB脉冲信号的长时间序列，FRB的波动与凸起可以被视作序列中的异常模式。为探索异常序列，本研究引入基于图的无监督时间序列异常检测方法，以应对FRB搜寻任务中“数据形态”层面的复杂性挑战。该方法不依赖标记样本，也无需异常的先验知识，能够适应不同长度的异常子序列。具体而言，首先将时间序列分割为子序列；随后通过一维卷积和主成分分析降维，将其投影到二维空间，在该空间中保留子序列形状的相似性；接着在二维空间中对各密集区域创建节点，构成图的顶点集合；最后使用滑动窗口在原始时间序列上移动，对任意相邻两个子序列，在嵌入空间中寻找各自的最近图顶点，遍历所有子序列以完成边的构造，并以出现频次作为边权。该过程在异常模式出现时会诱发“非常见”的顶点—顶点转移，从而以稀有路径或突变权重的形式被捕捉。快速射电暴FRB搜寻图化流程如图5所示。该方法已在FAST的真实历史观测数据中发现了数个被遗漏的微弱脉冲信号，验证了图方法在微弱信号搜寻方面的有效性与应用潜力。进一步地，该图式表示也便于在系统层采用流式处理、在硬件层利用高带宽并行加速，为后续“处理手段”与“计算模式”的优化留出接口与空间。

## 3.2. 高能物理实验中的径迹重建

高能物理是物理学最前沿的基础研究之一。借助粒子加速器，带电粒子被加速至极高速度，随后通过自发衰变或与目标物质相互作用产生新粒子，这些新粒子在探测器中留下命中信息。带电粒子的飞行径迹重建，对于分析粒子动量、电荷、粒子种类以及衰变或碰撞位置等信息至关重要。由于实验目标不同，径迹探测器在几何结构与探测原理上差异明显，因此需要结合具体装置进行有针对性的重建分析。近年来，机器学习在高能物理中得到广泛探索。例如在缪子反常磁矩实验中，现有重建算法通常需要对命中进行时间岛划分，再通过聚类与种子传播确定候选径迹命中，最后利用几何与误差传播算法进行拟合以得到动量与顶点信息。然而，该流程在聚类与种子传播阶段容易合并临近命中点，难以与串扰噪声区分，从而对最终重建结果产生影响。

为缓解上述问题，我们直接将命中点构建为图的节点，以命中点的位置信息和时间信息作为节点特征；结合探测器几何与物理约束建立边，边特征包括两节点空间差、时间差及其余弦相似度等；标注方面，将单个粒子在探测器上产生的任意两命中点之间的边标记为正边，其余为负边，从而把径迹重建转化为图上的边二分类与节点聚类联合问题。具体做法是，首先用图神经网络进行边预测，判断节点对之间是否存在飞行路径；随后根据边预测结果进行节点聚类，将同一粒子产生的全部命中点聚合，获得完整径迹。高能物理实验径迹重建图化流程如图6所示。相较于传统方法，该方案直接在原始命中层面建模，减少了聚类合并带来的偏差，提升了在复杂几何与噪声环境下的鲁棒性。这一工作凸显了图“处理手段”与科学任务适配的重要性，同时也体现了将科学数据转换为合适的图“数据形态”的必要性。

## 3.3. 核糖核酸二级结构相似性分析

核糖核酸（ribonucleic acid, RNA）存在于细胞与部分病毒、类病毒中。RNA的一级结构由腺嘌呤（A）、尿嘧啶（U）、鸟嘌呤（G）、胞嘧啶（C）四类碱基组成；二级结构由发卡环、茎、多连环等基本构件构成；当环内外碱基进一步作用形成假结点时，会出现更复杂的三级结构。一级结构的信息维度有限，难以反映碱基间空间作用关系；三级结构复杂且预测困难，因此信息较为完备且预测技术成熟的二级结构成为当前主要研究对象。基于RNA结构相似与功能相似之间的关联，通过分析二级结构相似性可发现与目标RNA功能相似的对象，为RNA功能发现与病毒治疗提供线索；反之亦可根据相似度差异进行RNA分类与新RNA发现。然而，现实工作中仍以肉眼对茎环结构进行主观判断为主，效率低且难以扩展；亦有曲线图形法、小波分析及其他生物学算法参与，但可解释性与结构信息捕获能力有限，面对海量基因库时如何高效筛选仍是难点。

为应对上述问题，我们将RNA二级结构自然转化为图结构：以碱基为图顶点、以互补碱基对为图边，从而把RNA之间的相似性分析问题转化为两个图之间的相似度计算问题。具体过程包括：首先进行图建模，获得RNA的图表示；为提高相似度评估的准确性与可解释性，进一步提出基于模式匹配的图算法，对RNA图中的各类基本构件（如发卡环、茎、多连环）进行识别与计数，并抽取构件之间的邻接关系；在此基础上构建高阶RNA图，实现从结构单元层面对二级结构进行可解释的对比。当面对大规模基因库时，可以采用并行化的图计算方法，将不同待对比RNA分配至多个计算节点与目标RNA同步进行相似度计算，在保持准确性的同时显著提升效率。RNA二级结构相似性分析图化流程如图7所示。这里重点需要考虑RNA结构图化过程中“数据形态”层面结构复杂、模式多样的挑战，同时在“处理手段”上要考虑方法的可解释性与自动化程度。

## 3.4. 药物间反应关系预测

药物间相互作用（drug-drug interaction, DDI）研究在药物安全性评估、临床用药指导以及老药新用等方面具有重要意义。通过预测潜在相互作用关系，可降低临床试错风险，提升研发效率，并推动个性化医疗发展。然而，药物种类繁多、潜在组合数量庞大；以DrugBank为例，每一百对药物中仅有一小部分存在相互作用，绝大多数相互作用仍依赖临床反馈被动发现。传统实验方法周期长、成本高，难以支撑大规模筛查。近年来的机器学习方法在一定程度上提高了效率，但仍存在明显局限：其一，依赖人工设计特征，难以充分捕捉药物间复杂的高阶交互；其二，依赖大量高质量标注样本，而现实中数据稀缺；其三，数据来源多样且存在噪声与不一致性，影响预测的可靠性与可解释性。

为解决这些问题，将图计算方法引入DDI预测。在统一的图框架下，将每种药物表示为图节点，特征由药物分子结构、分子指纹或靶点蛋白作用机制等组成；将已知或潜在相互作用表示为图边，并通过中继路径采样得到药物对在图中的最短反应路径，编码为边特征张量，以体现潜在作用通路。药物间反应关系预测图化流程如图8所示。基于此，本研究提出路径感知的孪生图神经网络：一方面通过图表示学习捕捉药物间的高阶依赖与上下文关系，另一方面结合图对比学习机制度量药物对的相似性与差异性，从而完成边预测并判断是否存在相互作用。该方法在有限标注条件下仍能有效建模，减少对人工特征与大规模标注的依赖；通过路径感知机制增强对潜在作用机制的捕捉，提高了可解释性；在工程实现上依托图计算框架可进行大规模并行训练，显著提升计算效率。这一应用实践，较好地体现了图计算作为一种新的“处理手段”在药物间反应关系预测领域中的应用潜力，并为“计算模式”层面的高并发执行提供了良好支撑。

## 3.5. 面向基因组图分析的图计算加速器

随着高通量测序技术的快速发展，基因分析的规模与复杂性持续攀升，传统依赖线性参考基因组的方法日益逼近吞吐瓶颈，难以满足大规模人群基因组研究与个性化医疗对实时性和精度的要求。相较之下，基因组图通过在图结构中显式表示基因的变异与复杂关系，能够更准确地反映个体或群体间的差异，被认为是下一代基因组学分析的重要方向。然而，基因组图分析的计算与存储需求远超传统方法：其一，访存敏感性突出，节点与边关系复杂导致候选比对序列的内存访问模式高度不规则、局部性差，大量时间消耗在内存访问；其二，计算敏感性显著，丰富的变异类型（如单核苷酸多态性、插入缺失、结构变异等）需要考虑多样化匹配模式，使计算量急剧上升；其三，负载异质性明显，播种阶段更依赖大规模索引操作，而对齐阶段涉及复杂比对与动态规划运算，二者在计算与存储特征上差异显著。

针对上述特征叠加造成的瓶颈，本研究提出基于图计算的新型加速方法，并在体系结构层面设计面向基因组图分析的处理架构[16]，如图9所示。核心思想是充分发挥近存计算与存算一体的优势，以应对高并发下的访存与计算需求。设计上采取非侵入式路径，确保与现有DRAM技术兼容，避免对主流存储硬件的重大改造，提升可部署性与演进性。具体机制包括：以数据驱动的图加速思路在硬件层面解耦复杂依赖，通过对图遍历与比对路径进行细粒度并行化，降低依赖链对吞吐的限制；在内存访问方面通过地址对齐与批量化访问降低不规则访存的开销，并引入哈希桶机制分簇播种序列，以提高索引阶段的吞吐；在对齐阶段，借助存算一体架构提供的高带宽支撑动态规划等计算密集型任务的加速执行。综合来看，该架构在保证结果准确性的同时显著提升了分析效率。这一工作重点解决了上述科学任务中“计算模式”层面关于负载均衡、能效与可扩展性的关键诉求，并与前述“数据形态—处理手段”的改进形成自洽闭环。

# 四、结束语

图计算驱动的科学计算Graph for Science通过图建模的方式把对象之间的依赖与关系直观呈现，从而突破了传统方法在复杂性、可扩展性与可解释性方面的局限。然而，由于普遍存在数据形态多样、处理方式复杂以及计算模式异质化等方面的问题，图计算驱动的科学计算仍面临诸多挑战。实践中，需要针对不同任务设计领域专用的构图策略和高效计算模型，这极大地影响了Graph for Science方法的效率和泛化能力。总体而言，Graph for Science为科学计算提供了新的思路和方法论，随着图计算理论与高性能计算机体系结构的不断演进，该范式有望在更多前沿科学问题中发挥作用，成为推动跨学科研究与技术创新的重要路径。而探索Graph for Science更深层的内在普遍一般规律，提高Graph for Science方法的效率和泛化能力也成为未来这一领域的核心关键问题之一，亟待我们去探索和实践。

**Graph Computing Accelerates Scientific Computing**

**Jin Hai**

**Abstract** Scientific data can often be directly or indirectly represented as graph structures with strong sparsity. As a crucial tool for analyzing complex relationships between entities, graph computing effectively supports the analysis of sparse relational data in scientific computing, enabling efficient processing of massive sparse datasets in this field. However, due to the complexity of scientific computing applications, graph-driven scientific computing faces challenges such as highly heterogeneous data forms, diverse processing methods, and difficulties in adapting computational models. To address these challenges, this paper investigates graph construction methods and corresponding graph computing techniques tailored for scientific computing across multiple domains. By leveraging graph computing technologies, we aim to efficiently meet the demands of various scientific computing applications. The proposed approach has been validated in several scientific computing fields, including fast radio burst detection, RNA secondary structure similarity analysis, and track reconstruction in high-energy physics experiments, exploring novel methodologies where graph computing provides solutions for scientific computing applications.

Key Words: sparse data processing; graph computing; scientific computing; graph construction method; domain-specific graph algorithm; acceleration system

摘要：科学计算数据通常能够直接或间接表示为图结构且具有较强的稀疏性。图计算作为分析事物之间复杂关联关系的重要工具，能够有效支持科学计算领域中数据间稀疏关联关系分析，实现科学计算领域中海量稀疏数据的高效处理。然而，由于科学计算应用的复杂性，图计算驱动的科学计算面临着数据形态纷繁芜杂、处理手段多样和计算模式难适配等挑战。为此，本文针对多个科学计算领域研究了面向科学计算的构图方法以及相应的图计算方法,通过图计算技术来高效支持各种科学计算应用的需求。通过在快速射电暴搜寻、RNA二级结构相似性分析以及高能物理实验径迹重建等多个科学计算领域进行了验证，探索了图计算为科学计算应用提供解决思路的新方法。
